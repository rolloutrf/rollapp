### Почему не Next.js?

RollApp — SPA, а не SSR-приложение. Причины:
- Финтех-приложение работает за авторизацией → SSR не даёт SEO-выгоды
- Чувствительные данные не должны рендериться на сервере
- React Native для мобильного клиента = общий код с SPA
- shadcn/ui работает одинаково с Vite и Next.js

### Почему shadcn/ui, а не Material UI / Ant Design?

| Критерий | shadcn/ui | MUI | Ant Design |
|----------|-----------|-----|------------|
| Владение кодом | ✅ Полное | ❌ node_modules | ❌ node_modules |
| Кастомизация | ✅ Редактируешь код | ⚠️ Theme override | ⚠️ ConfigProvider |
| AI-совместимость | ✅ Open code | ❌ Скомпилировано | ❌ Скомпилировано |
| Монорепо | ✅ Из коробки | ⚠️ Вручную | ⚠️ Вручную |
| Размер бандла | ✅ Tree-shake | ❌ Тяжёлый | ❌ Тяжёлый |
| Финтех-кастом | ✅ Полный контроль | ❌ Ограничен | ❌ Ограничен |

### Почему NestJS?

- Модульность → каждый микросервис — отдельный NestJS-модуль
- DI → тестируемость и подменяемость интеграций
- Guards → авторизация и валидация на уровне декораторов
- Interceptors → логирование, трансформация ответов
- Swagger/OpenAPI → автогенерация документации

## Мобильный клиент (будущее)

| Опция | Плюсы | Минусы |
|-------|-------|--------|
| **React Native** | Нативный UX, общий код с web | Два codebase (web + native) |
| **Capacitor** | Единый codebase, быстрый старт | Не нативный UX |
| **PWA** | Простейшая реализация | Ограничения платформ |

Рекомендация для MVP: **PWA** → затем **Capacitor** → затем **React Native** при необходимости.
